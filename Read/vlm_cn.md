# R³-Streaming：具有智能体控制的流式视频理解高效框架

**作者：** Jinming Liu, Jianguo Huang 等（上海交通大学 / 东方理工学院 / 微软亚洲研究院）· arXiv 2605.17921v2

---

## 1. 背景与动机 —— *为什么要做这项工作？*

**问题设定。** 离线视频大语言模型（字幕生成、摘要、长上下文问答）之所以效果好，是因为它们能一次性看到*整个*片段。但**流式**视频则不同：帧持续到达，每一个决策必须在*当下*做出，需满足严格的延迟预算，且无法预知未来。

**他们识别出的痛点/空白：**

- **静态的单点优化。** 现有的流式方法各自优化流水线中的*某一个*环节 — 仅优化响应触发，*或者*仅优化 Token/帧压缩。记忆保留、响应时机和推理深度之间从未进行*动态协调*。
- **被迫的权衡取舍。** 快速小模型在复杂查询上失败；"始终开启"的重模型既违反实时约束，又会在简单查询上*过度思考*。现有的决策-反应框架依赖单一反应模型 → 导致**效率-性能权衡不理想**。
- **缺乏对信息密度的自适应。** 真实流媒体的信息密度是变化的；固定策略无法适应。

**推动设计的两个实证发现（第3节，图1-2）：**

| 发现 | 证据 | 含义 |
|---|---|---|
| **F1：重要信号强烈集中于近期帧；历史 Token 冗余且甚至具有*误导性*。** | 在 OVO-Bench 上进行删除分析（通过下一 Token 分布的 Jensen-Shannon 散度衡量）：历史 Token 获得了*大部分*视觉注意力，然而删除它们几乎不改变输出 — 删除*邻近* Token 导致约 1.7 倍的更大变化。激进压缩历史（Nearby=1.0, Historical=0.01）使 StreamingBench 分数*提升* **+2.4**（71.0 vs 无压缩）。 | → 驱动了 **Remember（主动遗忘）**：高保真保留近期上下文，激进遗忘旧上下文。 |
| **F2：模型规模的增益在流式任务上并非单调递增。** | Qwen2.5-VL-**3B** 在实时/前瞻任务上优于 **7B**；Qwen3-VL-4B-Thinking ≈ 8B-Thinking。"始终开启"的重推理既昂贵又*有时效果更差*。 | → 驱动了 **Reason（自适应思考）**：仅将困难查询路由至慢模型。 |

> **幻灯片要点：** 他们将流式理解从*被动的逐帧前馈*任务重新构建为一个**智能体控制问题** — 智能体管理自身状态并选择自身的计算路径。

## 2. 核心创新 —— *有什么新意？*

**R³-Streaming = Remember（记忆）, Respond（响应）, Reason（推理）** — 一个**级联控制框架**，将每个流式步骤分解为三个顺序耦合的决策，使得每个下游决策都建立在逐步细化的信息状态之上：

1. **Remember（主动遗忘）** — 一个*无需训练*的、基于时效感知的记忆压缩器。近期帧以高保真保留，陈旧帧被整合到紧凑槽位中。
2. **Respond（主动响应）** — 一个轻量级*准备就绪判定头*，当证据不足时发出 `<Routine>` 信号以**推迟**回答（避免过早产生幻觉）。
3. **Reason（自适应思考）** — 一个*路由策略*：快速模型要么直接回答（`<Answer>`），要么发出 `<Escalate>` 信号调用更重的推理/思考模型。

**两个主要技术贡献：**

- **时效感知遗忘策略** — 经验洞察：*激进压缩历史可以提升准确率*（而不仅仅是节省计算），由此转化为双区记忆调度方案。
- **TB-GRPO（目标均衡 GRPO）** — 一种强化学习目标函数，**稳定化二值路由策略**，并将升级率保持在*可部署*的计算预算内。这是关键创新：普通 GRPO 和 AutoThink 会崩溃为"始终升级"；TB-GRPO 对升级频率增加了显式的**目标区间控制**。

**动作空间：** `A = {<Answer>, <Escalate>, <Routine>}`。

> **为什么这很重要：** 在 StreamingBench 上，自适应路由器*优于*直接纯慢模型推理 — 所以收益**不是**简单的固定快/慢混合；智能体控制真正做到了为每个查询选择更优的动作。

## 3. 方法论 —— *如何构建的？*

### 3.1 问题建模
在步骤 *t*：观察流历史 `x₁:ₜ`，接收查询 `qₜ`，维护记忆 `Mₜ`，然后进行**级联**决策。由于每个决策都消费前一个决策的输出，**错误会累积**（噪声记忆 → 不良准备就绪 → 不必要的升级）— 这正是他们将三者联合协调的原因。

### 3.2 Remember（主动遗忘）—— *无需训练*
将历史划分为一个**邻近**窗口（大小 *W*）和一个**历史**区域，用不同的阈值进行压缩：

$$M_t = \text{Compress}(x_{t-W+1:t},\ \tau_{near}) \cup \text{Compress}(x_{1:t-W},\ \tau_{hist})$$

其中 **τ_near ≫ τ_hist**（近 = 细粒度 Token；远 = 紧凑的情节槽位）。
- 压缩算子：**DTD**（来自 TimeChat-Online）。已被证明对算子选择具有鲁棒性（DivPrune/VisionZip/平均池化均可行）— *时效感知策略*比算子本身更关键。
- 默认设置（流式）：邻近阈值 **1.0**（无压缩），历史 **0.01**，邻近窗口 **3 帧** → **95–96% Token 丢弃率**。
- 离线长视频：历史重置为 **0.5**（约 45% 丢弃）— 这是一个确定性的*场景切换*，而非逐视频搜索，因为部署时先验地知道流是实时的还是预录的。

### 3.3 Respond（主动响应）
一个准备就绪判定头 *h* 估算可回答性：

$$p_{ready} = h(q_t, M_t), \qquad a_t = \begin{cases} \text{发出 } \texttt{<Routine>}, & p_{ready} < 0.5 \\ \text{继续进入 Reason}, & \text{否则} \end{cases}$$

### 3.4 Reason（自适应思考）—— 最难的部分
二值路由：直接回答 vs `<Escalate>`。分**两个阶段**学习（SFT 冷启动 → TB-GRPO；详见下文第5节）。

**TB-GRPO 机制。** 对一个查询，策略采样一组 `{yᵢ}`；`eᵢ = 1[yᵢ = <Escalate>]`，`cᵢ` = 正确性，`ρ = (1/G)Σeᵢ` = 组内升级比例。

*基础奖励*（编码"有能力则直接回答，必要时则升级"）：

$$r^{naive}_i = \begin{cases} 2 & e_i{=}0, c_i{=}1 \;\text{（正确的直接回答 — 最优，延迟最低）}\\ -1 & e_i{=}0, c_i{=}0\\ 1 & e_i{=}1, c_i{=}1\;\text{（正确但较慢）}\\ 0 & e_i{=}1, c_i{=}0 \end{cases}$$

*围绕操作点 (η, γ) 的目标区间惩罚：*

$$\delta_{esc} = \text{clip}(\rho - (\eta+\gamma), 0, 1), \qquad \delta_{ans} = \text{clip}((\eta-\gamma) - \rho, 0, 1)$$

- `ρ > η+γ`（过度升级）→ δ_esc>0 抑制升级奖励
- `ρ < η−γ`（升级不足）→ δ_ans>0 惩罚直接回答
- 在区间内 → 两者均为零（对路由频率的比例/反馈控制）

这些调节基础奖励 `rᵢ`，然后使用标准**组归一化优势** `Aᵢ = (rᵢ − r̄)/(std+ε)` 与**裁剪 GRPO + KL** 目标：

$$\mathcal{L}_{TB\text{-}GRPO} = \mathbb{E}[\min(w_i A_i, \text{clip}(w_i, 1-\epsilon_c, 1+\epsilon_c)A_i)] - \beta_{KL} D_{KL}(\pi_\theta \| \pi_{ref})$$

设置：**η = 0.3, γ = 0.2**。(η, γ) 对是计算预算的一个直接、可调的控制旋钮。

> **训练动力学证据（图6）：** 普通 GRPO 在第40步崩溃至 ρ=1.0；AutoThink 避免了早期崩溃但收敛到高位；**TB-GRPO 收敛到更低、更稳定的比例*同时*获得更高奖励。**

## 4. 实验 —— *效果如何？*

**设定。** Fast = Qwen2.5-VL-3B/7B；Slow = Qwen3-VL-4B/8B-Thinking 或 Qwen2.5-VL-32B。记法 `R3-Streaming-[Fast]|[Slow]`。

**主要流式结果（流式 MLLM 中的 SOTA）：**

| 基准测试 | 最佳配置 | 分数 | 备注 |
|---|---|---|---|
| **OVO-Bench** | 7B \| 4B-Thinking | **57.92** | 96% Token 缩减；超越 Streamo-7B (55.61), StreamAgent-7B (49.4), TimeChat-Online (45.6) |
| **StreamingBench** | 7B \| 4B-Thinking | **76.36** | 95% Token 缩减；超越 StreamAgent (74.28), Dispider (67.63)，甚至 GPT-4o (73.28) |

值得注意的是，它**同时超越了**自身慢模型的基线（Qwen3-VL-4B-Thinking, 57.74/73.16）和快模型的基线（Qwen2.5-VL-7B）→ 收益不是固定的快/慢混合。

**向离线长视频的泛化（表3）：** 使用 Historical=0.5，R3-Streaming-3B|4B = **MLVU 70.6 / Video-MME 65.5**，击败 AdaReTaKe、VideoAgent、TimeChat-Online、StreamAgent。

**关键消融实验：**

- **模块具有互补性（表4）：** 基线 3B → +Remember (52.8) → +Reason (55.9) → **+Both = 56.6**（OVO 基准）；StreamingBench 趋势相同。
- **Remember（表5）：** 时效感知 DTD = **75.90** & 95% 丢弃 vs DivPrune/VisionZip/DTD 单独使用 (65–68%)。收益来自*策略*，而非算子。
- **Respond（表6）：** 专用准备就绪头 = **0.328** 主动输出 vs 0.216 (3B) / 0.204 (7B) — 来自学习，而非模型规模。
- **Reason 路由（表7）：** SFT-only 和普通 GRPO → 100% 升级（崩溃）；AutoThink → 53.2% 升级率，70.0 准确率；**TB-GRPO → 74.36 准确率，仅 24.0% 升级率。**
- **(η, γ) 扫描（表8）：** 显式的准确率↔升级率控制，例如 η=0.3, γ=0.2 → 74.4 准确率 / 24% 升级率。
- **效率（图7）：** 自适应路由在*所有*慢模型上以远低于每帧延迟的代价超越纯慢模型。
- **记忆-路由协同（附录D.3）：** 恢复 Nearby=1.0 将整体准确率从 64.72% 提升至 73.64%，*同时*升级率从 37.04% 下降至 24.04% — 更好的近期上下文使快速路径能够自足。路由适应任务难度：物体感知约 20% 升级率；计数 49–72%，前瞻推理 >65%。

## 5. 训练数据构建与训练顺序 *（您特别询问的内容）*

**各模块的总体训练顺序：**

```
① Remember  → 无需训练（无参数；仅配置切换）
② Reason 第一阶段：SFT 冷启动  （教授路由*格式*）
③ Reason 第二阶段：TB-GRPO 强化学习      （教授路由*决策质量*）
④ Respond：准备就绪头 SFT     （最后训练，在 ②+③ 完成后）
```
Respond 被特意安排在**最后**训练，且**快 VLM 冻结** — 仅更新轻量级准备就绪头 — 这样就不会干扰已优化好的快/慢路由器。

### 各模块的数据构建

**② Reason — SFT 冷启动数据（图4，附录D.1）。** 基于 **TimeChat-Online-139K** 构建。
- 对每个查询，快模型采样 **K = 4** 条回复。
- **开放式问题：** 由外部大语言模型（**Qwen3-14B**）在 **5 分制**上比较回复与真实答案进行评分。
- **客观问题：** 二元精确正确性 `s ∈ {0,1}`。
- 聚合 `s̄ = (1/K)Σs`，然后标注：`s̄ ≥ T → <Answer>`，否则 `<Escalate>`。阈值 **T = 2.5**。
- 快模型在这些混合目标上微调 — 这仅修正*格式*（"学会说路由语言"），而非决策质量。

**③ Reason — TB-GRPO 强化学习数据。** 同样来自 TimeChat-Online-139K；奖励由正确性加上述目标区间规则在线计算（η=0.3, γ=0.2）。这细化了*何时*升级的决策。

**④ Respond — 准备就绪 SFT 数据（附录C）。** 基于 **TimeChat-Online-139K + COIN** 构建，使用**以决策边界为中心的难例挖掘**而非随机采样：
- 识别问题首次变得可回答的精确**线索时间戳 t_c**。
- **未就绪 → `<Routine>`：** 线索出现*前*的 3 帧，`{t_c−3, t_c−2, t_c−1}`。
- **就绪 → 进入 Reason：** 线索帧 + 后续 2 帧，`{t_c, t_c+1, t_c+2}`。
- 边界裁剪到有效范围；故意*省略*此窄窗口之外的帧。这将监督信号集中到刚好之前/之后的决策边界 → 学会避免过早产生幻觉，同时最小化反应延迟。

## 6. 结论与局限性

**洞见/贡献：**
- 流式理解最好建模为**级联智能体控制**，而非前馈推理。
- **遗忘历史是主动有益的**（噪声去除），而不仅仅是压缩妥协。
- **选择性推理**（TB-GRPO）在准确率*和*延迟上均优于始终开启的重模型，且带有可调预算旋钮。
- SOTA 流式结果 + 强大的离线泛化 + 95–96% Token 缩减。

**局限性（论文陈述 + 个人解读）：**
- **最坏情况延迟尖峰：** TB-GRPO 仅约束*平均*升级比例。在持续高信息密度片段期间，即使快模型也难以应对，升级可能聚集 → 突发尖峰仍可能超出预算。当前基准测试仅衡量*平均值*；作者呼吁开发**尖峰感知基准测试**。
- *（您可在讨论环节提出的额外角度）：* Respond 使用固定 0.5 阈值；SFT 标签依赖大语言模型评分器（Qwen3-14B）的质量；在线/离线历史阈值是手动切换而非学习的；路由是二值的（没有多层级"中等"模型）；仅在 Qwen 系列骨干模型上评估。

**与您工作的相关性：** Remember 模块本质上是一个*物体/证据持久记忆*问题 — 基于 JSD 的发现表明*近期* Token 占主导而历史容易误导注意力，这与您的持久物体记忆方向（您特别希望跨长间隙保留被遮挡物体的状态）构成有趣的对照。他们的时效感知遗忘恰好会丢弃您项目需要保留的证据，如果这将融入您的 VLM 物体状态记忆项目，这是一个值得在幻灯片中展示的有趣张力点。
